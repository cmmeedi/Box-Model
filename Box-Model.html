Box Model

In this section I learn about the Box model and how to design boxes using styles such as:
  • Border
  • Padding
  • Margin
  • Display
  • Units

What I Learned

  • Border

    • Borders are the lines around a box. These lines can be styled differently by changing up the Width, Color, Style, and Radius.

  • Padding

    • Within these borders are content whether it be an image or text within a p element. The space between this content and the borders is called padding and that space can be enlarged or minimized in accordance to how the user wants.

  • Margin

    • As Padding does for the space within an elements borders, Margin does the same for the outside of an elements borders.

  • Display

    • Elements can be changed from either Inline to Block with the Display property.

Border
  • Border Width

    • Border width will determine how thick the walls of a box(border) will be

  • Border Style

    • The style of the border can be among many but a few exmaples you can see around each of the sections of within this site. You can see a Ridge style, Double Style, Solid Style and None style.

  • Border Color

    • The border color will change what color is assigned to the border

  • Border Shorthand

    • border: 3px solid black; This is the shorthand for styling borders. From left to right it assigns the border width, then the style then the color.

  • Border Radius

    • The corners of borders can be rounded by adjusting the border radius. You can see the borders on some of these sections are rounded and this was done by using border radius.

Padding and Margin

  • Padding

    • Padding is the space between an elements border and the content within the borders

  • Padding All Around

    • padding: 50px; /*(for example)*/

    Will apply 50px of padding to each side of the content within the borders of an element

  • Padding Vertical/Horizontal

    • padding: 50px 25px; /*(for example)*/

    Will apply 50px of padding to the top and bottom and will apply 25px to the left and right of the content within the elements borders

  • Padding Top/Horizontal/Bottom

    • padding: 25px 30px 70px;

    will apply 25px to the top, 30px to the left and the right, and 70px to the bottom of the content within an elements border

  • Padding(Shorthand) Top/Right/Bottom/Left

    • The shorthand property of Padding allows you to assign the padding to each side of the content using 1 style declaration

    padding: 25px, 20px, 10px, 70px;

    the convention for this is clockwise starting from the top so 25px top, 20px right, 10px bottom, 70px left.

  • Margin

    • Margin is the space between two separate elements

  • Margin All Around

    • margin: 50px; /*(for example)*/

    Will apply 50px of margin to the outside of an elements borders

  • Margin Vertical/Horizontal

    • margin: 50px 25px; /*(for example)*/

    Will apply 50px of margin to the top and bottom and will apply 25px to the left and right of the outside of an elements borders

  • Margin Top/Horizontal/Bottom

    • margin: 25px 30px 70px;

    will apply 25px to the top, 30px to the left and the right, and 70px to the bottom of the outside of an elements borders

  • Margin(Shorthand) Top/Right/Bottom/Left

    • The shorthand property of Padding allows you to assign the margin to each side of the content using 1 style declaration

    margin: 25px, 20px, 10px, 70px;

    the convention for this is clockwise starting from the top so 25px top, 20px right, 10px bottom, 70px left.

Display and Units

  • Display

  • Inline

    • Inline will take all the elements selected and place them side by side

  • Block

    • Block display will assign a whole Block(row) for the selected element

  • Inline-Block

    • Inline-Block will allow you to place block oriented elements side by side but also allow certain styles like 'Margin' and 'Padding' to take effect.

  • Units

  • px

    • Pixels are referring to Pixels like those on your screen. These are small units and are generally used when making small adjustments (or at least should be).

  • Percentages

    • Percentages are in reference to whatever element that your selector is assigned to and will reference that elements size to give an exact percentage proportion

  • em

    • Referring to the parent element, the font size you set with 'em' is that of the parent element. 3em is 3 times the size of the parent element

    The 'em' can also refer to the size of the element within itself. Kind of confusing but whatever the text element size is, within the selector the 'em' will refer to the 'em' within the parent of the current selector.

    Here is an example:
    body{
    font-size: 100px;
    }
    main{
    font-size: 1em; /* This font size is equal to 100px */
    }
    div{
    font-size: .5em; /* This font size is equal to 50px */
    }
    p{
    font-size: 1em; /* This font size is equal to 50px */
    }

    NOTE: if this is too confusing to understand stick with the description above until it is understood

    Because em always refer to the parent element this can cause some unforseen madness like with nested lists. The size can drastically change rather quickly due to parent elements being created without thought.

  • REM

    • or Root em. Rem will only and alway refer to the root element when sizes are being adjusted. This can prevent the above mentioned "madness" from happening since there is only one reference to size

    If no root element is set then by default whatever the font size of the 'HTML' tag at the very top of the HTML document is will be the root size.
